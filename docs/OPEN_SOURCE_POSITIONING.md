# Open-source positioning

TSAO aims to be an open process-development method and interoperability layer. It does not claim ownership of underlying chemical science, standards, third-party software or trademarks.

The repository favours small, inspectable scientific kernels, explicit schemas and adapter contracts. Commercial tools may participate without becoming the canonical record. Contributions should be independently implementable, license-compatible and accompanied by provenance and tests.

“Most powerful” is treated as a measurable engineering aspiration: breadth of supported decisions, depth of executable contracts, failure detection, reproducibility, evidence quality and honest limitation reporting—not promotional language or file count.
