# Maintainer release checklist

- [ ] Scope and affected Gates are explicit.
- [ ] Evidence and license provenance are current.
- [ ] Schemas and migrations are tested.
- [ ] Scientific kernels have known-solution and failure tests.
- [ ] Project end-to-end initialization and audit pass.
- [ ] No technical approval is overstated.
- [ ] Two CI rounds produce the same core identity.
- [ ] Archive builds are byte-identical.
- [ ] Cleanroom extraction and verification pass.
- [ ] Release notes identify external work and rollback conditions.
