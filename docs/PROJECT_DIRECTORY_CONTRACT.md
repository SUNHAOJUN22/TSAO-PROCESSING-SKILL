# Project directory contract

A TSAO project uses numbered lifecycle directories from governance and evidence through field learning, plus raw/processed data, models, reports and logs. The exact directory names are created by `tsao init` and validated by `tsao audit`.

The project manifest is the machine contract. It records project identity, selected domains and subskills, G0–G18 Gate records and the technical approval boundary. Initial projects must state `technical_approval_status: NOT_EVALUATED`.

Directory presence proves only structural initialization. Technical completeness requires evidence-bearing artifacts, qualified models, acceptance results and accountable approval.
