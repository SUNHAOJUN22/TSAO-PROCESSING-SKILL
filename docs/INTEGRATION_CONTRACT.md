# Specialist integration contract

A specialist skill inherits the TSAO master lifecycle, Gate states, evidence schema, model-risk classes, status boundary, project contract and release process.

It contributes domain ontology, route alternatives, scientific models, process-specific failure modes, templates and tests. A specialist may add Gates or stricter criteria, but cannot remove master requirements, weaken fail-closed behaviour or promote historical case data to approved design basis.

Integration is accepted only when master routing, project initialization, specialist entrypoint, inherited tests, migration checks and approval-boundary attacks pass.
