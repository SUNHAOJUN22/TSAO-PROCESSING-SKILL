# Governance

TSAO is governed by evidence, review and reproducibility rather than file count.

- Maintainers control releases, schemas and security response.
- Domain leads own specialist decision logic and scientific tests.
- Gate approvers must be named and independent of the author for MR4/MR5 or high-consequence decisions.
- Changes to Gate semantics, schemas, approval boundaries or release verification require a design note and migration test.
- No contributor may mark real experiments, FTO, HAZOP/LOPA/SIL, customer qualification or industrial performance as complete without auditable evidence and accountable approval.
- Releases use semantic versioning, deterministic archives, manifests, checksums and cleanroom verification.
