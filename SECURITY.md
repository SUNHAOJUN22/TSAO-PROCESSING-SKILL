# Security Policy

Report vulnerabilities privately through GitHub Security Advisories. Do not place plant credentials, proprietary process data, customer data, API keys, simulator licenses or export-controlled information in public issues.

Supported releases are the latest tagged release and the active development branch. Maintainers will triage injection, path-traversal, archive, secret-leakage, dependency, workflow-permission and unsafe-code-execution issues.

TSAO-generated safety material is a planning and handoff artifact, not a substitute for qualified process-safety review. Security or safety controls may not be disabled to make CI pass.
