# TSAO Process Intelligence OS

## Mission

Use TSAO to develop, evaluate, model, scale up, troubleshoot, retrofit, optimize, qualify or transfer a chemical process or technology package. Route generic chemistry to `process-general`, EPM/EPDM to `epdm`, POE solution polymerization to `poe`, and other polymer work to `polymer-general`, while retaining this master governance layer.

A complete invocation must produce auditable project artifacts rather than generic advice.

## Non-negotiable rules

1. Start with decisions, boundaries, owners and acceptance criteria.
2. Separate observed, reported, calculated, inferred, assumed, proposed, planned, rejected and approved states.
3. Record source locators, dates, applicability, contradictions and review expiry.
4. Never copy a literature, patent, supplier or inherited-case number into a production basis without project validation.
5. Qualify every model for purpose, risk, equations, parameters, units, data lineage, identifiability, uncertainty, applicability domain and independent review.
6. State preserved physics, broken similarity, compensating evidence and rollback criteria for every scale-up claim.
7. Gates are fail-closed; missing evidence means `HOLD` or `NOT_EVALUATED`.
8. Software qualification is separate from scientific, engineering, safety, legal, customer and industrial approval.
9. HAZOP/LOPA/SIL, FTO, customer trials, pilot operation and plant guarantees remain accountable external work.
10. Produce reusable files and machine-readable records.

## Mandatory one-call execution

1. classify project mode, chemistry, phase, operation, product form, limiting physics and hazards;
2. inventory supplied files and build source, data, conflict, assumption and gap registers;
3. create falsifiable questions, competing routes, experiments, models and acceptance criteria;
4. instantiate G0–G18 across fourteen professional workstreams;
5. generate 266 fail-closed work packages, M0–M9 maturity and explicit external-execution status;
6. close mass, element, energy, impurity, utility, water, VOC and carbon balances;
7. perform integrity review before experiment freeze and package release;
8. publish decision, science, process, model, scale-up, package and acceptance artifacts;
9. keep unexecuted experiments, commercial models and unsigned safety reviews as `PLANNED` or `REQUIRES_EXTERNAL_EXECUTION`.

## G0–G18

G0 governance; G1 evidence/IP; G2 product/CQA; G3 measurement/raw materials/data; G4 chemistry/catalyst; G5 mechanism/kinetics/properties; G6 laboratory feasibility; G7 model qualification; G8 continuous bench/dynamics; G9 separation/recovery; G10 pilot basis; G11 pilot execution/model update; G12 demonstration; G13 industrial process/equipment/utilities; G14 control/digital twin/transition; G15 HSE/reliability/TEA/LCA/supply/IP; G16 product/customer/regulatory qualification; G17 package freeze/commissioning/performance test; G18 post-commercial monitoring/MOC/learning.

Allowed status: `NOT_EVALUATED`, `HOLD`, `CONDITIONAL`, `PASS`, `FAIL`, `RETIRED`.

## Fourteen workstreams

Governance; evidence/IP; product/quality; chemistry; analytics/data; statistics/DoE; kinetics/properties; reactor/multiphysics; process synthesis; scale-up/pilot; control/operations; HSE/reliability; TEA/supply/IP; reporting/package/transfer.

## Maturity

M0 idea; M1 evidence-framed; M2 chemistry-proven; M3 laboratory-process-proven; M4 bench-integrated; M5 pilot-ready; M6 pilot-proven; M7 demo-proven; M8 industrial-ready; M9 operationally-validated. Documents alone never advance maturity.

## Health and release

Run `tsao doctor` before project work and before release. The complete ZIP carries full specialist source trees and per-file provenance. The public source tree publishes exact parity status and must never imply full sync when it is partial.
